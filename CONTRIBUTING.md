# 贡献与反馈指南

这个仓库目前主要由作者维护，但也长期欢迎外部贡献。为了减少来回沟通、提升处理效率，请在提交 Issue 或 PR 前先阅读本页。

## 提交 Issue 前先确认

1. 先阅读 [README.md](./README.md) 中的安装说明、广告说明和常见问题排查。
2. 确认脚本管理器已经启用，并已开启“允许用户脚本”或同类权限。
3. 尽量先切换其他解析源测试一次，排除单个第三方解析源临时失效。
4. 如果是移动端问题，请优先说明浏览器名称、系统版本和脚本权限状态。

## 选择正确的 Issue 模板

- `Bug report（Bug 反馈）`
  - 用于脚本本身的异常、功能回归、按钮不显示、逻辑错误等问题。
- `Feature request（功能建议）`
  - 用于新增功能、交互优化、体验改进或维护建议。
- `Parser or site report（站点兼容 / 解析源异常反馈）`
  - 用于某视频网站无法解析、某解析源失效、自动解析异常、移动端兼容问题等。

## 高质量反馈的最低要求

请尽量一次性提供完整信息，否则 Issue 很可能无法处理：

- 完整的视频页面链接，而不是站点首页
- 具体的视频网站名称
- 浏览器与版本
- 脚本管理器与版本
- 操作系统与设备类型
- 可复现的操作步骤
- 截图、录屏或控制台报错
- 如果是解析问题，请说明具体解析源名称

## 这些情况不一定属于脚本 Bug

以下问题经常出现，但未必是脚本自身逻辑错误：

- 单个第三方解析源临时失效、打不开或速度很慢
- 第三方解析页自带广告、跳转、伪装按钮
- 视频清晰度差、卡顿、音画不同步
- 站点自身改版或临时风控导致的短期异常

这类问题仍然可以反馈，但请把“具体站点、链接、解析源、环境信息”写清楚，否则很难定位。

## Pull Request 规则

欢迎提交 PR，但请尽量遵守以下约定：

1. 改动前先确认问题背景，必要时先开 Issue 说明。
2. PR 标题和内容请尽量聚焦单一问题，不要一次混入多个无关改动。
3. 按仓库提供的 PR 模板填写改动背景、解决方案、兼容性与测试结果。
4. 如果改动影响移动端、站点兼容或解析源逻辑，请在描述中明确写出影响范围。
5. 不要顺手重构与当前问题无关的大段代码，除非它直接阻碍本次修改。

## 自动维护策略

仓库已启用基础自动维护：

- 新建 Issue 后会自动回复一次，提醒补充必要信息。
- 长时间没有新活动的 Issue 或 PR 会被自动标记为 `stale`。
- 在被标记后仍长期无更新的 Issue 或 PR，会被自动关闭。

如果你的 Issue 或 PR 仍然有效，只需要补充信息、回复进展或推送新提交，`stale` 标记就会被移除。

## 最后说明

Issue 和 PR 模板不是为了增加门槛，而是为了减少“信息不全导致无法处理”的情况。你提供的信息越完整，问题通常越容易被快速定位。
